# IQBook: архитектура формата и приложений на Unity

Этот документ фиксирует техническую концепцию гибрида gamebook/visual novel/интерактивного учебника в защищённом формате `.iqbook`.

## 1) Формат `.iqbook`

Рекомендуемый контейнер: ZIP с фиксированной структурой:

```text
metadata.json       – название, автор, версия формата, хэш контента
signature.bin       – цифровая подпись (RSA) всех файлов контента
content.json        – граф истории (узлы, выборы, связи с медиа)
assets/
  images/           – иллюстрации (png/jpg)
  videos/           – видеофрагменты (mp4)
  maps/             – файлы описания интерактивных карт (json + изображение)
```

Ключевые свойства:
- интерактивные карты через JSON (изображение + зоны переходов);
- медиа (изображения/видео) через ссылки внутри `assets/`;
- ветвление через `choices` и условия по флагам.

### Пример `content.json`

```json
{
  "nodes": [
    {
      "id": "start",
      "type": "dialogue",
      "text": "Вы стоите перед замком.",
      "illustration": "assets/images/castle.png",
      "video": null,
      "map": null,
      "choices": [
        { "text": "Войти", "next_node": "hall" },
        { "text": "Уйти", "next_node": "forest" }
      ]
    },
    {
      "id": "hall",
      "type": "map",
      "map": "assets/maps/hall.json",
      "choices": []
    }
  ]
}
```

### Пример файла карты `hall.json`

```json
{
  "image": "assets/images/hall_map.png",
  "zones": [
    { "rect": [100, 200, 300, 400], "node": "throne_room" },
    { "rect": [500, 100, 700, 300], "node": "kitchen" }
  ]
}
```

## 2) Разделение на приложения

- **Редактор**: создание графа, ресурсы, предпросмотр, экспорт `.iqbook`.
- **Читалка**: только воспроизведение и прохождение.

Преимущества: меньше поверхности атаки, изоляция функционала подписи/редактирования.

## 3) Архитектура Unity

### Общая библиотека (Shared)
- модели: `Node`, `Choice`, `MapData`;
- упаковка/распаковка `.iqbook` (ZIP + опциональное шифрование);
- проверка подписи и хэшей.

### Редактор
- граф сюжета на GraphView (`UnityEditor.Experimental.GraphView`) или через ScriptableObject;
- импорт/менеджмент ресурсов;
- экспорт: сборка архива, хэш, подпись приватным ключом.

### Читалка
- загрузка и валидация `.iqbook`;
- UI (uGUI/UI Toolkit), рендер текста, выборов, иллюстраций и видео;
- карты как экран с зонами перехода;
- локальные сохранения прогресса (текущий узел + флаги).

## 4) Защита контента и авторских прав

### Базовый стек
- SHA-256 для хэшей;
- RSA для цифровой подписи;
- AES + PBKDF2 (`Rfc2898DeriveBytes`) для шифрования приватного ключа паролем.

### Создание книги (Редактор)
1. Автор задаёт пароль.
2. Генерируется пара RSA-ключей.
3. Приватный ключ шифруется паролем (AES/PBKDF2).
4. Публичный ключ сохраняется в метаданных.
5. Контент хэшируется, хэш подписывается приватным ключом.
6. Подпись (`signature.bin`) добавляется в архив.

### Открытие в Читалке
- проверка подписи публичным ключом;
- при несовпадении подписи — отказ в загрузке.

### Открытие в Редакторе
- запрашивается пароль для восстановления приватного ключа;
- без приватного ключа переподписать изменения нельзя, поэтому редактирование запрещается (доступен read-only режим).

## 5) Практические рекомендации

- хранить рабочий проект автора в обычной структуре папок/JSON;
- экспорт в `.iqbook` делать финальным шагом;
- для карт использовать нормализованные координаты зон (`0..1`) для адаптивного UI.

## 6) Медиа и производительность

- видео воспроизводить через `VideoPlayer` и путь к локальному файлу;
- тяжёлые видео при необходимости хранить рядом с книгой (внешние ресурсы), но включать их в схему подписи;
- избегать загрузки больших видео в память целиком.

## 7) Дорожная карта MVP → Production

1. Схемы JSON для графа/карт/метаданных.
2. Библиотека упаковки/распаковки + подпись.
3. MVP Читалки: текст, выборы, изображения.
4. Поддержка видео и интерактивных карт.
5. MVP Редактора: граф + ресурсы + экспорт.
6. Пароли/ключи/подписи.
7. Кроссплатформенное тестирование.
